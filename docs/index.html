Features

Everything an agent — or a developer — needs to read a Confluence wiki.

⚡

Fast & Focused

Partial page reads — outline, section or keyword — so an agent loads only the context it needs.

🔍

Powerful Search

Search with raw CQL or simple filter flags: text, author, space, label, type and date ranges.

💬

Comments & More

Read page comments and post replies — the one write operation, kept deliberately explicit.

🔒

Secure by Default

Credentials live in the OS keychain with a 0600 file fallback, never in the config file.

🤖

Agent-First Output

JSON by default, structured errors with exit codes and recovery hints a coding agent can act on.

☁️

Cloud & Data Center

One flavor-agnostic client; the backend is detected automatically.

Install

Pick whichever method suits you.

# npm
npm install -g @angelmsger/confluence-cli

# go
go install github.com/angelmsger/confluence-cli/cmd/confluence-cli@latest

# from source
make install

Or download a prebuilt binary from the Releases page.

Shell completion (optional)

Completes subcommands, flag values and live space keys. Load it once:

source <(confluence-cli completion bash)

Companion Skill

A confluence Skill is embedded in the binary — it detects your coding agent (Claude Code, Codex) and deploys for each one found:

confluence-cli skill install

Quick start

Configure once, then read your wiki.

# interactive setup, then verify connectivity
confluence-cli config init
confluence-cli doctor

# search, then read a page in stages
confluence-cli search --text "release process"
confluence-cli page get <id|url> --scope outline
confluence-cli page get <id|url> --scope section --section sec-2
confluence-cli comment list <id|url>
